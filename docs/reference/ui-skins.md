# UI Skins Reference

UI skins are local ZIP packages that customize Construct's dashboard with
validated design tokens and local image assets. v1 skins are intentionally not
plugins: they cannot run JavaScript, ship arbitrary CSS, embed HTML, use SVG, or
load remote URLs.

## ZIP Structure

Each package must include `construct-skin.json` at the ZIP root.

```text
construct-skin.json
assets/
  logo.png
  avatar.webp
  hero.jpg
  shell-texture.png
  panel-decoration.webp
```

Only these file extensions are accepted:

- `.json`
- `.png`
- `.jpg`
- `.jpeg`
- `.webp`

Assets must live under `assets/`. The gateway stores imported skins under:

```text
{workspace_dir}/skins/{skin_id}/manifest.json
{workspace_dir}/skins/{skin_id}/assets/...
```

## Manifest Schema

```json
{
  "schemaVersion": 1,
  "id": "rabbit_garden",
  "name": "Rabbit Garden",
  "version": "1.0.0",
  "modes": {
    "light": {
      "tokens": {
        "--construct-bg-base": "#f7fbf2",
        "--construct-bg-surface": "#ffffff",
        "--construct-text-primary": "#142018",
        "--construct-signal-live": "#4e9f5b",
        "--construct-radius-md": "14px"
      },
      "assets": {
        "brandLogo": "assets/logo.png",
        "operatorAvatar": "assets/avatar.webp",
        "dashboardHero": "assets/hero.jpg",
        "shellTexture": "assets/shell-texture.png",
        "panelDecoration": "assets/panel-decoration.webp"
      },
      "preview": "assets/hero.jpg"
    },
    "dark": {
      "tokens": {
        "--construct-bg-base": "#090607",
        "--construct-bg-surface": "#151012",
        "--construct-text-primary": "#f6ecef",
        "--construct-signal-live": "#f08aaa"
      },
      "assets": {
        "brandLogo": "assets/logo.png",
        "operatorAvatar": "assets/avatar.webp"
      },
      "preview": "assets/hero.jpg"
    }
  }
}
```

Required manifest fields:

| Field | Purpose |
|---|---|
| `schemaVersion` | Must be `1` |
| `id` | Storage ID; lowercase letters, numbers, `-`, and `_` only |
| `name` | Display name |
| `version` | Skin package version |
| `modes` | Object containing `light`, `dark`, or both |

Supported modes are `modes.light` and `modes.dark`. Each mode may define
`tokens`, `assets`, and `preview`.

## Token Contract

Skin-authored tokens must use the canonical `--construct-*` namespace.
Existing `--pc-*` compatibility tokens are generated by the frontend bridge and
must not be authored directly.

Accepted token values:

- color values: hex, `rgb()`, `rgba()`, `hsl()`, `hsla()`
- bounded CSS lengths for radius, spacing, and font-size-like tokens: `px`,
  `rem`, `em`, or `%`

Rejected in v1:

- `url(...)`
- `var(...)`
- `expression(...)`
- JavaScript-like URLs
- shadow tokens
- background-image tokens
- arbitrary CSS snippets

Common useful tokens:

| Token | Use |
|---|---|
| `--construct-bg-base` | page background |
| `--construct-bg-shell` | sidebar/shell surface |
| `--construct-bg-surface` | cards and contained surfaces |
| `--construct-bg-panel` | translucent panel background |
| `--construct-border-soft` | low-emphasis borders |
| `--construct-border-strong` | active/selected borders |
| `--construct-text-primary` | primary text |
| `--construct-text-secondary` | secondary text |
| `--construct-text-muted` | muted labels |
| `--construct-signal-live` | primary accent |
| `--construct-signal-network` | secondary accent |
| `--construct-radius-sm` | compact radius |
| `--construct-radius-md` | panel radius |
| `--construct-radius-lg` | large radius |

## Asset Slots

| Slot | UI use |
|---|---|
| `brandLogo` | sidebar/header brand image |
| `operatorAvatar` | operator/agent avatar surfaces |
| `dashboardHero` | dashboard visual hero |
| `shellTexture` | subtle shell background texture |
| `panelDecoration` | subtle panel corner decoration |

Asset values must be relative paths under `assets/`, for example
`assets/logo.webp`. Remote URLs, absolute paths, path traversal, and SVG are
rejected.

## Gateway API

Routes:

| Route | Purpose |
|---|---|
| `GET /api/skins` | list installed skins |
| `POST /api/skins/import` | import a ZIP package |
| `DELETE /api/skins/{id}` | delete an installed skin |
| `GET /api/skins/{id}/assets/{path}` | serve a stored skin asset |

Mutation routes require normal gateway authentication. The import route has its
own upload limit and timeout:

| Limit | Value |
|---|---|
| ZIP body | 25 MiB |
| Request timeout | 120 seconds |
| Extracted total size | 50 MiB |
| File count | 128 |

Asset serving returns a content type, `X-Content-Type-Options: nosniff`, and
private cache headers.

## Validation Rules

The importer rejects packages that contain:

- missing or duplicate root `construct-skin.json`
- absolute paths
- `..` path traversal
- path separators that escape the skin root
- symlinks
- duplicate ZIP entries
- unsupported extensions
- unsupported modes or asset slots
- invalid skin IDs
- manifest asset references that are missing from the ZIP
- JavaScript, arbitrary CSS, HTML, SVG, remote URLs, or executable behavior

Deleting a skin removes only `{workspace_dir}/skins/{skin_id}` after canonical
path verification.
